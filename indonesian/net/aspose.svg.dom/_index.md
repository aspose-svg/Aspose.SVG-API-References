Itu **Aspose.Svg.Dom (Model Objek Dokumen)** namespace menyediakan API yang mewakili dan berinteraksi dengan dokumen HTML, XML, atau SVG apa pun. DOM adalah model dokumen yang dimuat di browser dan mewakili dokumen sebagai pohon simpul, di mana setiap node mewakili bagian dari dokumen (misalnya elemen, teks string, atau komentar).

## Kelas

| Kelas | Keterangan |
| --- | --- |
| [Attr](./attr/) | Antarmuka Attr mewakili atribut dalam objek Elemen. Biasanya nilai yang diizinkan untuk atribut ditentukan dalam skema yang terkait dengan dokumen. |
| [CDATASection](./cdatasection/) | Bagian CDATA digunakan untuk keluar dari blok teks yang berisi karakter yang seharusnya dianggap sebagai markup. |
| [CharacterData](./characterdata/) | CharacterData memperluas Node dengan seperangkat atribut dan metode untuk mengakses data karakter di DOM. |
| [Comment](./comment/) | Mewarisi dari CharacterData dan mewakili konten komentar, yaitu, semua karakter di antara awalan ' . |
| [Document](./document/) | Dokumen mewakili seluruh dokumen HTML, XML atau SVG. Secara konseptual, ini adalah akar pohon dokumen, dan menyediakan akses utama ke data dokumen. |
| [DocumentFragment](./documentfragment/) | DocumentFragment adalah objek Dokumen "ringan" atau "minimal". Sangat umum jika ingin dapat mengekstraksi sebagian pohon dokumen atau membuat fragmen baru dari dokumen. |
| [DocumentType](./documenttype/) | DocumentType menyediakan antarmuka ke daftar entitas yang ditentukan untuk document |
| [DOMException](./domexception/) | Antarmuka DOMException mewakili kejadian abnormal (disebut pengecualian) yang terjadi sebagai akibat dari pemanggilan metode atau pengaksesan properti API web. Ini pada dasarnya adalah bagaimana kondisi kesalahan dijelaskan dalam API web. |
| [DOMObject](./domobject/) | Tipe DOMObject digunakan untuk mewakili objek dasar untuk seluruh Model Objek Dokumen. Untuk Java dan ECMAScript, DOMObject terikat ke tipe Objek. |
| [Element](./element/) | Antarmuka Elemen mewakili elemen dalam dokumen HTML atau XML. |
| [Entity](./entity/) | Mewakili entitas yang dikenal, baik diurai atau tidak diurai, dalam dokumen XML. |
| [EntityReference](./entityreference/) | node EntityReference dapat digunakan untuk mewakili referensi entitas di pohon. |
| [EventTarget](./eventtarget/) | Itu[`EventTarget`](../aspose.svg.dom/eventtarget/) antarmuka diimplementasikan oleh semua Node dalam implementasi yang mendukung DOM Event Model. Oleh karena itu, antarmuka ini dapat diperoleh dengan menggunakan metode pengecoran khusus pengikatan pada instance antarmuka Node. Antarmuka memungkinkan pendaftaran dan penghapusan Pendengar Acara di sebuah[`EventTarget`](../aspose.svg.dom/eventtarget/) dan pengiriman acara untuk itu[`IEventTarget`](../aspose.svg.dom.events/ieventtarget/) . |
| [Node](./node/) | Antarmuka Node adalah tipe data utama untuk seluruh Model objek Dokumen. Ini mewakili satu simpul di pohon dokumen. |
| [Notation](./notation/) | Merupakan notasi yang dideklarasikan dalam DTD. |
| [ProcessingInstruction](./processinginstruction/) | ProcessingInstruction merepresentasikan "instruksi pemrosesan", yang digunakan dalam XML sebagai cara untuk menyimpan informasi khusus prosesor dalam teks dokumen. |
| [ShadowRoot](./shadowroot/) | ShadowRoot adalah simpul akar dari pohon bayangan. |
| [Text](./text/) | Antarmuka Teks mewarisi dari CharacterData dan mewakili konten tekstual (disebut data karakter dalam XML) dari Elemen atau Attr. |
| [TypeInfo](./typeinfo/) | TypeInfo merepresentasikan tipe yang direferensikan dari node Element atau Attr, ditentukan dalam skema yang terkait dengan dokumen. |
